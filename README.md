# power-control

电源控制项目仓库，用于管理电源开关、供电状态检测、异常保护和上位机/下位机通信。

当前仓库先作为项目骨架，后续可以按硬件板卡、单片机固件、上位机控制程序和测试文档逐步补全。

## 项目目标

- 控制多路电源输出的开关状态。
- 采集电压、电流、温度等关键状态量。
- 提供过流、过压、欠压、过温等保护逻辑。
- 支持串口、CAN、ROS 或其他上位机接口扩展。
- 保留清晰的硬件、固件、上位机和文档分工。

## 仓库结构

```text
.
├── docs/       # 需求、协议、测试记录和使用文档
├── firmware/   # 单片机/嵌入式固件代码
├── hardware/   # 原理图、PCB、BOM、接口定义
├── host/       # 上位机工具、调试脚本或 ROS 节点
└── README.md
```

## 初步模块划分

| 模块 | 内容 |
| :--- | :--- |
| 硬件层 | 电源输入、DC-DC、MOS/继电器开关、电流/电压采样、保护电路。 |
| 固件层 | GPIO/PWM/ADC 采集、状态机、保护逻辑、通信协议解析。 |
| 上位机 | 串口/CAN 调试工具、状态显示、开关控制、日志记录。 |
| 文档 | 接口定义、通信协议、测试步骤、故障处理记录。 |

## 开发约定

- 不提交真实密钥、个人隐私信息或仅本机可用的绝对路径。
- 硬件接口、协议帧格式和测试结果优先写入 `docs/`。
- 固件和上位机代码尽量保留最小可运行示例。
- 重要改动通过 issue 或 pull request 讨论，方便团队协作。
